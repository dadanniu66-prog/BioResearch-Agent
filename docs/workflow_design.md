# 工作流设计：BioResearch-Agent

## 1. 项目总体架构

BioResearch-Agent 是一个面向医学科研训练的多 Agent 协作系统，目标是帮助医学本科生和科研初学者完成从公共数据库理解、数据整理、R 语言分析、结果解释到科研报告生成的完整流程。

它不是一个简单的“AI 写代码工具”，而是一个可以参与科研流程拆解、代码生成、错误诊断和结果解释的医学科研智能体系统。

整体工作流如下：

```text
用户输入科研问题
        ↓
数据库理解 Agent
        ↓
数据预处理 Agent
        ↓
R 代码生成 Agent
        ↓
错误诊断与结果检查 Agent
        ↓
生物医学结果解释 Agent
        ↓
科研报告生成 Agent
```

## 2. 用户输入

用户可以输入一个医学科研想法，例如：

```text
我想研究非酒精性脂肪肝中蛋白修饰相关基因的作用，
希望结合 GEO 数据集、差异分析、WGCNA、机器学习和单细胞验证。
```

系统会把这个比较模糊的科研想法拆解成可以执行的分析步骤。

例如，一个原始想法：

```text
我想做非酒肝相关的蛋白修饰基因研究。
```

可以被拆解为：

```text
1. 明确疾病方向：非酒精性脂肪肝 / NASH / MASH
2. 搜索合适的 GEO 数据集
3. 判断数据类型：芯片、bulk RNA-seq 或单细胞数据
4. 整理表达矩阵和样本分组
5. 进行差异表达分析
6. 进行 WGCNA 共表达网络分析
7. 结合蛋白修饰相关基因集筛选候选基因
8. 使用机器学习进一步筛选关键基因
9. 进行富集分析和机制解释
10. 使用单细胞数据验证关键基因的细胞来源
11. 生成科研报告和图表解释
```

## 3. 数据库理解 Agent

数据库理解 Agent 负责读取和理解公共数据库信息，例如 GEO、NHANES、单细胞数据库等。

主要任务包括：

- 判断数据集类型：芯片、bulk RNA-seq、单细胞 RNA-seq、空间转录组或流行病学队列数据
- 识别样本分组：疾病组、对照组、治疗组、不同时间点等
- 判断数据文件类型：原始 count 矩阵、表达矩阵、平台注释文件、临床信息文件等
- 判断是否适合后续分析：差异分析、WGCNA、机器学习、单细胞验证等
- 提醒潜在问题：样本量太小、分组不清楚、基因注释缺失、批次效应等

示例输出：

```text
数据集编号：GSE126848
数据类型：转录组表达数据
主要任务：表达矩阵整理和样本分组构建
可进行的分析：差异表达分析、WGCNA、富集分析、机器学习筛选
潜在问题：需要进一步确认样本分组和基因注释方式
```

## 4. 数据预处理 Agent

数据预处理 Agent 负责把原始数据整理成后续分析可以使用的标准格式。

主要任务包括：

- 读取原始表达矩阵
- 检查基因 ID 或 gene symbol
- 去除空基因名
- 合并重复基因
- 整理样本名称
- 构建样本分组表
- 输出标准化后的表达矩阵和分组文件

预期输出文件包括：

```text
expression_matrix_clean.csv
sample_group.csv
gene_annotation.csv
analysis_log.txt
```

其中：

```text
expression_matrix_clean.csv
```

用于保存整理后的表达矩阵，一般要求行为基因，列为样本。

```text
sample_group.csv
```

用于保存样本分组信息，例如每个样本属于 disease 组还是 control 组。

```text
analysis_log.txt
```

用于记录本次数据整理过程中遇到的问题和处理方式，方便后续复现。

## 5. R 代码生成 Agent

R 代码生成 Agent 负责根据具体科研任务生成对应的 R 脚本。

支持的分析模块包括：

- GEO 表达矩阵整理
- 差异表达分析
- WGCNA 共表达网络分析
- 机器学习筛选关键基因
- GO / KEGG / GSEA 富集分析
- ROC 曲线分析
- 免疫浸润分析
- 单细胞数据验证
- 结果可视化

示例脚本模块：

```text
01_prepare_GEO_expression_matrix.R
02_differential_analysis.R
03_wgcna_pipeline.R
04_machine_learning_screening.R
05_enrichment_analysis.R
```

R 代码生成 Agent 不只是简单生成代码，还需要根据数据类型选择合适的方法。

例如：

```text
芯片数据：优先考虑 limma
bulk RNA-seq count 数据：优先考虑 DESeq2 或 edgeR
已经标准化的表达矩阵：需要先判断是否适合直接进行差异分析
单细胞数据：可使用 Seurat 进行质控、降维、聚类和注释
```

## 6. 错误诊断与结果检查 Agent

错误诊断 Agent 负责读取 R 语言运行日志、终端输出和报错信息，判断分析是否成功。

主要任务包括：

- 识别 R 包安装错误
- 识别文件路径错误
- 识别表达矩阵维度异常
- 识别分组信息不匹配
- 检查输出文件是否生成
- 根据报错信息给出下一步修改建议

例如：

```text
如果 expression_matrix_clean.csv 已经生成，
并且行是基因、列是样本，则说明表达矩阵整理步骤基本成功。

如果 sample_group.csv 已经生成，
并且包含 sample_id 和 group 两列，则说明样本分组文件已经初步完成。

如果 R 报错显示 file not found，
则优先检查当前工作目录、文件夹名称和文件路径是否一致。
```

这个 Agent 的价值在于帮助初学者从“看到报错就懵”转变为“知道下一步该检查哪里”。

## 7. 生物医学结果解释 Agent

生物医学结果解释 Agent 负责把生信分析结果转化成医学科研语言。

主要任务包括：

- 解释差异基因说明了什么
- 解释 WGCNA 模块和疾病表型的关系
- 解释机器学习筛选出的关键基因为什么重要
- 解释富集分析中的通路含义
- 解释单细胞结果和 bulk 结果如何互相验证
- 帮助用户形成“每张图讲什么”的科研故事线

示例输出：

```text
Figure 1：展示数据集来源、样本分组和整体研究流程。
Figure 2：展示疾病组与对照组之间的差异表达基因。
Figure 3：通过 WGCNA 筛选与疾病表型相关的关键模块。
Figure 4：使用机器学习进一步筛选候选诊断基因。
Figure 5：通过单细胞数据验证关键基因在哪些细胞类型中高表达。
```

这个 Agent 主要解决一个常见问题：

```text
很多学生会跑代码、会出图，但是不知道这些图合在一起到底讲了一个什么科研故事。
```

## 8. 科研报告生成 Agent

科研报告生成 Agent 负责把分析结果整理成可以用于申报、汇报或论文写作的材料。

支持生成的内容包括：

- 大创项目申报书
- 课题设计方案
- 论文初稿框架
- 图注说明
- 结果解释
- 汇报 PPT 大纲
- 给导师看的研究进展总结

例如，当用户完成了差异分析、WGCNA 和机器学习后，报告 Agent 可以帮助整理成：

```text
研究背景
研究目的
数据来源
研究方法
结果设计
创新点
可行性分析
预期成果
```

## 9. 一个完整示例流程

以 GEO 数据集为例，一个完整流程如下：

```text
确定研究方向
    ↓
搜索 GEO 数据集
    ↓
读取数据集说明
    ↓
下载表达矩阵和样本信息
    ↓
整理表达矩阵
    ↓
构建样本分组表
    ↓
进行差异表达分析
    ↓
进行 WGCNA 分析
    ↓
差异基因与关键模块基因取交集
    ↓
使用机器学习筛选核心基因
    ↓
进行富集分析和通路解释
    ↓
在单细胞数据中验证核心基因
    ↓
生成科研报告和图表解释
```

## 10. 多 Agent 协作逻辑

BioResearch-Agent 的核心不是单个模型一次性完成所有任务，而是将复杂科研流程拆分给不同 Agent。

整体协作方式如下：

```text
数据库理解 Agent：
负责判断数据能不能用、怎么分组、适合什么分析。

数据预处理 Agent：
负责把数据整理成标准表达矩阵和分组文件。

R 代码生成 Agent：
负责生成可以运行的 R 脚本。

错误诊断 Agent：
负责检查代码运行结果和报错信息。

生物医学解释 Agent：
负责解释结果和疾病机制之间的关系。

科研报告 Agent：
负责把结果整理成申报书、论文框架或汇报材料。
```

这种设计可以让整个项目更加可复现，也更适合医学本科生学习。

## 11. 后续扩展计划

后续计划包括：

- 自动解析 GEO 数据集
- 自动判断芯片数据和测序数据的处理方式
- 自动构建发现集和验证集
- 增加 NHANES 生存分析模块
- 增加孟德尔随机化模块
- 增加网络药理学模块
- 增加单细胞细胞注释辅助模块
- 增加中文教学解释模式
- 增加项目长期记忆和版本追踪功能

## 12. 项目意义

BioResearch-Agent 的意义在于降低医学本科生进入 AI + 生物信息学科研的门槛。

很多低年级医学生并不是没有科研兴趣，而是不知道如何从一个模糊的研究想法走到可执行的数据分析流程。本项目希望通过 AI Agent 的方式，把数据库理解、代码生成、结果解释和科研写作串联起来，形成一个可复现、可学习、可扩展的医学科研训练系统。
