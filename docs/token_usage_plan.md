# Token 使用计划：BioResearch-Agent

## 1. 为什么本项目需要大量 Token

BioResearch-Agent 是一个面向医学科研训练的 AI Agent 项目，它不是简单的聊天机器人，也不是只生成几行代码的工具。

本项目的核心任务是让 AI 参与完整的医学科研流程，包括：

- 阅读公共数据库说明文档
- 理解 GEO / NHANES / 单细胞数据结构
- 生成 R 语言分析脚本
- 读取和诊断 R 语言报错日志
- 根据输出结果判断分析是否成功
- 解释生物信息学图表
- 生成科研申报书、论文框架和汇报材料
- 支持多个 Agent 之间的长上下文协作

这些任务都需要处理大量文本、代码、日志和科研解释内容，因此对 token 的消耗非常高。

## 2. 主要 Token 消耗场景

### 2.1 公共数据库理解

在医学科研中，一个数据集通常包含大量说明信息，例如：

- 数据集标题
- 样本来源
- 疾病类型
- 平台信息
- 原始文件
- 表达矩阵
- 样本分组
- 临床信息
- 补充材料

AI Agent 需要读取这些信息，并判断数据是否适合后续分析。

例如，用户输入一个 GEO 数据集编号后，Agent 需要完成：

```text
读取数据集说明
    ↓
判断数据类型
    ↓
识别样本分组
    ↓
判断是否有表达矩阵
    ↓
判断是否需要平台注释
    ↓
推荐差异分析、WGCNA、机器学习或单细胞验证方案
```

这个过程需要较长上下文支持。

## 3. R 代码生成与修改

本项目需要 AI 根据不同数据类型生成 R 语言代码。

常见任务包括：

- 表达矩阵读取
- 基因 ID 转换
- 去除空基因名
- 合并重复基因
- 构建样本分组
- 差异表达分析
- WGCNA 分析
- LASSO 回归
- 随机森林
- ROC 曲线
- 富集分析
- 单细胞 Seurat 分析

这些代码不是一次生成就结束，而是需要多轮修改。

例如：

```text
第一轮：生成初始 R 脚本
第二轮：根据用户文件结构修改路径
第三轮：根据报错修改代码
第四轮：检查输出文件是否正确
第五轮：解释结果并生成下一步分析代码
```

每一轮都需要保留前文上下文，因此 token 消耗明显高于普通问答。

## 4. 报错日志诊断

医学数据分析中经常会出现复杂报错，例如：

- R 包没有安装
- Bioconductor 版本不匹配
- 文件路径错误
- 文件名不一致
- 表达矩阵格式错误
- 样本名和分组表不匹配
- gene symbol 缺失
- 矩阵维度异常
- 单细胞对象过大导致内存不足

AI Agent 需要读取用户提供的终端日志、R 控制台输出和项目文件结构，然后判断问题发生在哪里。

示例流程：

```text
用户上传报错截图或日志
        ↓
错误诊断 Agent 读取报错内容
        ↓
判断是路径问题、包问题还是数据格式问题
        ↓
给出具体修改方案
        ↓
用户重新运行代码
        ↓
Agent 再次检查输出结果
```

这个过程通常需要多轮对话，因此非常消耗 token。

## 5. 多 Agent 协作带来的 Token 消耗

BioResearch-Agent 的设计不是单 Agent，而是多 Agent 协作。

主要 Agent 包括：

```text
数据库理解 Agent
数据预处理 Agent
R 代码生成 Agent
错误诊断 Agent
生物医学解释 Agent
科研报告生成 Agent
```

每个 Agent 都需要读取前一个 Agent 的输出，并生成自己的结果。

例如：

```text
数据库理解 Agent 输出数据集信息
        ↓
数据预处理 Agent 根据数据集信息整理矩阵
        ↓
R 代码生成 Agent 根据矩阵结构生成代码
        ↓
错误诊断 Agent 根据运行日志修改代码
        ↓
结果解释 Agent 根据输出图表解释生物学意义
        ↓
科研报告 Agent 整理成申报书或论文框架
```

这种连续协作模式需要较长上下文窗口和大量 token 支持。

## 6. 生物医学结果解释

医学科研项目不仅需要代码运行成功，还需要解释结果。

例如，差异分析、WGCNA、机器学习和单细胞分析分别回答不同问题：

```text
差异分析：哪些基因在疾病组和对照组之间表达不同？
WGCNA：哪些基因模块和疾病表型相关？
机器学习：哪些基因更适合用于分类或预测？
单细胞分析：关键基因主要在哪些细胞类型中表达？
富集分析：这些基因可能参与哪些生物学通路？
```

AI Agent 需要把这些结果串联成科研故事线，例如：

```text
疾病发生
    ↓
关键通路异常
    ↓
候选基因筛选
    ↓
机器学习验证
    ↓
单细胞定位
    ↓
潜在机制解释
```

这类解释通常需要较长文本和多轮修订，因此也是重要 token 消耗来源。

## 7. 科研文书生成

本项目还需要生成多种科研文书，包括：

- 大创项目申报书
- 课题设计方案
- 研究背景
- 技术路线
- 创新点
- 可行性分析
- 预期成果
- 论文初稿框架
- 图注说明
- 汇报 PPT 大纲

这些文书通常篇幅较长，并且需要结合前面的数据分析结果进行修改。

例如，AI 需要把分析流程整理成：

```text
研究背景
研究目的
研究内容
技术路线
创新点
可行性分析
预期成果
```

这类长文本生成和多轮润色也会产生大量 token 消耗。

## 8. 计划中的 Token 使用方向

如果获得 token 支持，本项目计划主要用于以下方向：

| 使用方向 | 具体用途 |
|---|---|
| 数据集解析 | 读取 GEO、NHANES、单细胞数据说明 |
| 代码生成 | 生成 R 语言分析脚本 |
| 代码调试 | 读取 R 报错日志并修复代码 |
| 工作流解释 | 向医学本科生解释每一步为什么这样做 |
| 结果解释 | 解释差异分析、WGCNA、机器学习和单细胞结果 |
| 科研写作 | 生成申报书、论文框架和汇报材料 |
| 多 Agent 协作 | 测试不同 Agent 之间的任务传递 |
| 长上下文记忆 | 保持一个项目从选题到结果解释的连续性 |

## 9. 预计使用案例

### 9.1 GEO 数据集分析

用户输入：

```text
我想用 GSE126848 做非酒精性脂肪肝相关分析。
```

Agent 需要完成：

```text
解析数据集
    ↓
判断数据类型
    ↓
整理表达矩阵
    ↓
构建分组文件
    ↓
生成差异分析代码
    ↓
生成 WGCNA 代码
    ↓
生成机器学习筛选代码
    ↓
解释结果
    ↓
生成报告
```

### 9.2 NHANES 队列分析

用户输入：

```text
我想研究饮食炎症指数和癌症幸存者预后的关系。
```

Agent 需要完成：

```text
读取 NHANES 变量说明
    ↓
选择暴露变量、结局变量和协变量
    ↓
合并人口学、饮食、疾病和死亡数据
    ↓
构建分析队列
    ↓
进行加权回归分析
    ↓
进行亚组分析和敏感性分析
    ↓
生成论文结构
```

### 9.3 单细胞分析

用户输入：

```text
我想验证关键基因主要在哪些细胞类型中表达。
```

Agent 需要完成：

```text
读取单细胞数据
    ↓
质量控制
    ↓
标准化
    ↓
降维聚类
    ↓
细胞类型注释
    ↓
关键基因表达展示
    ↓
解释关键基因和细胞类型的关系
```

## 10. 预期影响力

获得 token 支持后，BioResearch-Agent 预计可以产生以下影响：

1. 降低医学本科生学习生物信息学的门槛。
2. 提高公共数据库分析流程的可复现性。
3. 减少初学者在 R 语言报错和路径问题上浪费的时间。
4. 帮助学生理解每张科研图背后的逻辑。
5. 将 AI Agent 应用于医学科研训练和教学场景。
6. 为后续大创项目、科研训练和论文写作提供可复用模板。

## 11. 总结

BioResearch-Agent 是一个高 token 消耗的 AI Agent 项目。

它需要长期处理公共数据库说明、R 语言代码、终端日志、科研图表解释和长篇科研文书，并通过多 Agent 协作完成完整医学科研流程。

因此，本项目非常适合使用大规模 token 支持进行持续开发、测试和优化。
