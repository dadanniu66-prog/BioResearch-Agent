# BioResearch-Agent

## 项目简介

BioResearch-Agent 是一个面向医学本科生和科研初学者的 AI 驱动生物医学科研工作流智能体项目。

本项目希望利用 AI Agent 帮助用户完成从公共数据库理解、数据整理、R 语言代码生成、运行报错诊断、结果解释到科研报告生成的完整流程。

项目重点面向以下医学科研场景：

- GEO 公共转录组数据分析
- NHANES 队列数据分析
- bulk RNA-seq 分析
- WGCNA 共表达网络分析
- 机器学习筛选关键基因
- 单细胞数据验证
- 蛋白修饰相关基因集研究
- 大创申报书、开题报告和论文框架生成

本项目目前处于原型设计和工作流验证阶段，主要用于探索 AI Agent 在医学科研训练和生物信息学学习中的应用。

---

## 项目背景

医学科研正在快速进入数据驱动阶段。GEO、NHANES、单细胞数据库等公共资源中包含大量可用于疾病机制研究、标志物筛选和临床预测的数据。

但是，对于医学本科生和科研初学者来说，完整完成一个生物信息学项目并不容易。

一个完整项目通常需要同时掌握：

```text
公共数据库检索
表达矩阵整理
样本分组构建
R 语言编程
差异表达分析
WGCNA
机器学习
富集分析
单细胞分析
科研图表解释
论文式报告写作
```

这些步骤学习成本较高，而且很容易卡在文件路径、R 包安装、矩阵格式、分组信息和结果解释等问题上。

BioResearch-Agent 的目标就是通过多 Agent 协作，把复杂科研流程拆解成可学习、可执行、可复现的步骤。

---

## 核心痛点

本项目主要解决以下问题：

1. 医学本科生不知道如何判断一个 GEO / NHANES 数据集是否适合分析。
2. 初学者很容易卡在表达矩阵整理、样本分组和基因注释阶段。
3. R 语言运行过程中经常出现路径错误、包安装错误和矩阵格式错误。
4. 很多学生会跑出图，但不知道每张图在科研逻辑中说明什么。
5. 生物信息学分析流程碎片化，缺少可复现的项目结构。
6. 从分析结果过渡到大创申报书、开题报告或论文框架时缺少系统辅助。

---

## 项目目标

BioResearch-Agent 计划构建一个多 Agent 协作系统，帮助医学科研初学者完成以下任务：

- 理解公共数据库
- 判断数据类型和样本分组
- 生成 R 语言分析脚本
- 诊断代码运行报错
- 检查输出文件是否正确
- 解释差异分析、WGCNA、机器学习和单细胞结果
- 生成科研报告、图表解释和项目申报材料

---

## 多 Agent 工作流

BioResearch-Agent 的核心设计是多 Agent 协作，而不是单纯让 AI 生成几段代码。

整体工作流如下：

```text
用户输入科研问题
        ↓
数据库理解 Agent
        ↓
数据预处理 Agent
        ↓
R 代码生成 Agent
        ↓
错误诊断与结果检查 Agent
        ↓
生物医学结果解释 Agent
        ↓
科研报告生成 Agent
```

---

## Agent 模块说明

### 1. 数据库理解 Agent

负责读取和理解 GEO、NHANES、单细胞数据库等公共数据来源。

主要任务：

- 判断数据集类型
- 识别样本分组
- 判断文件类型
- 判断是否适合差异分析、WGCNA、机器学习或单细胞验证
- 提醒样本量不足、分组不清、批次效应等潜在问题

---

### 2. 数据预处理 Agent

负责把原始数据整理成后续分析可用的标准格式。

主要任务：

- 读取表达矩阵
- 整理 gene symbol 或 gene ID
- 去除空基因名
- 合并重复基因
- 构建样本分组表
- 输出标准表达矩阵和分组文件

---

### 3. R 代码生成 Agent

负责根据具体任务生成 R 语言脚本。

支持模块：

- GEO 表达矩阵整理
- 差异表达分析
- WGCNA 分析
- LASSO / 随机森林等机器学习筛选
- GO / KEGG / GSEA 富集分析
- ROC 曲线分析
- 单细胞 Seurat 分析
- 图表可视化

---

### 4. 错误诊断 Agent

负责读取 R 语言运行日志、终端输出和报错信息。

主要任务：

- 判断 R 包是否缺失
- 判断文件路径是否错误
- 判断表达矩阵格式是否正确
- 判断样本名和分组表是否匹配
- 检查输出文件是否生成
- 给出下一步修改建议

---

### 5. 生物医学结果解释 Agent

负责把生信分析结果转化成医学科研语言。

主要任务：

- 解释差异基因结果
- 解释 WGCNA 模块和疾病表型的关系
- 解释机器学习筛选出的关键基因
- 解释富集分析中的通路含义
- 解释单细胞结果中关键基因的细胞来源
- 形成 figure-by-figure 的科研故事线

---

### 6. 科研报告生成 Agent

负责把分析结果整理成科研文书。

支持生成：

- 大创项目申报书
- 课题设计方案
- 论文初稿框架
- 研究背景
- 技术路线
- 创新点
- 图注说明
- 汇报 PPT 大纲

## 为什么需要大量 Token 支持

本项目是一个高 token 消耗场景。

原因包括：

1. 需要读取较长的 GEO、NHANES 和单细胞数据说明。
2. 需要生成和修改较长的 R 语言分析脚本。
3. 需要读取 R 控制台报错、终端日志和输出文件信息。
4. 需要进行多轮代码调试和结果检查。
5. 需要解释复杂的生物信息学图表和医学机制。
6. 需要生成大创申报书、开题报告、论文框架等长文本。
7. 多 Agent 协作需要保留较长上下文。

因此，BioResearch-Agent 非常适合使用大规模 token 支持进行持续开发、测试和优化。

## 项目意义

BioResearch-Agent 希望降低医学本科生进入 AI + 生物信息学科研的门槛。

很多初学者并不是没有科研兴趣，而是不知道如何从一个模糊的选题走到完整的数据分析流程。本项目希望通过 AI Agent，把数据库理解、代码生成、结果解释和科研写作连接起来，形成一个可复现、可学习、可扩展的医学科研训练系统。
